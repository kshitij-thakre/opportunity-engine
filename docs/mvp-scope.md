# Opportunity Engine - MVP Scope Boundaries

This document defines what is inside and outside the scope of the initial Minimum Viable Product (MVP).

## In Scope
1. **Single Source Ingestion**: Initial focus on pulling opportunity listings from a single platform (e.g., Upwork or a public bidding board).
2. **Dashboard Feed**: Clean, sortable table and detail cards for viewing fetched opportunities.
3. **Manual Filtering & Tagging**: Basic controls for users to mark opportunities as "Qualified", "Interesting", or "Hidden".
4. **Basic Search & Categorization**: Keywords and tags search within the local database.

## Out of Scope
1. **AI / LLM Integrations**: Generative proposal text writing or machine-learning-based classification are deferred to post-MVP.
2. **Automatic Bid Submission**: Bidding must be done manually on target sites; the tool acts only as a discovery and pipeline helper.
3. **CRM Synchronization**: Integrated sales management pipelines (Salesforce, HubSpot) are deferred.
4. **Multi-User Teams**: Authentication support will be configured for single accounts/owners initially.
